
# Customer Behaviour Analysis – Data Analytics Project

## Project Overview
This project focuses on analyzing customer behaviour in a banking dataset using Python. The goal is to understand spending patterns, engagement levels, and customer segmentation to extract meaningful business insights.

The dataset contains 120,000+ records, making it suitable for real-world data analysis practice.

## Objectives
- Analyze customer spending patterns
- Understand relationship between transactions, logins, and spending
- Identify customer segments based on behaviour
- Detect patterns related to complaints and account balance
- Extract business insights from raw data

## Dataset Information
The dataset includes the following features:
- Customer_ID, User_ID
- Account_Type, Home_Branch
- Transactions, Total_Spend
- Logins, Complaints
- Account_Balance
- Date

## Tools & Technologies Used
- Python
- Pandas
- Matplotlib
- Seaborn
- Jupyter Notebook / Google Colab

## Key Analysis Performed
- Data cleaning and preprocessing
- Exploratory Data Analysis (EDA)
- Spending distribution analysis
- Logins vs spending relationship
- Transactions vs spending analysis
- Customer segmentation based on spending
- Complaints impact analysis

## Key Insights
- Customers can be grouped into Low, Medium, and High spending segments
- Higher engagement (logins & transactions) is associated with higher spending
- A small group of customers contributes significantly to total revenue
- Complaints may indicate reduced customer engagement
- Data patterns help in understanding customer behaviour for business decisions

## Project Outcome
This project helped in understanding how raw banking data can be transformed into meaningful insights using data analysis techniques.

It demonstrates skills in:
